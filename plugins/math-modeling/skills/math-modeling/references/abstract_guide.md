# 竞赛论文摘要生成指南

## 摘要的重要性

摘要是竞赛论文中权重最高的单页。评委通常在 15-30 秒内通过摘要决定论文是否入围。摘要的质量直接决定最终获奖等级。

## 摘要结构模板

摘要应包含以下要素，按顺序组织：

1. **问题重述**（1-2 句）：用自己的话概述问题的核心
2. **方法概述**（2-3 句）：简述采用的建模方法和创新点
3. **核心数值结果**（必须包含具体数字）：关键定量结论
4. **创新点**（1 句）：本工作最突出的贡献
5. **结论**（1-2 句）：总结性评价和建议

### 美赛/MCM/ICM 英文摘要
- 长度：200-300 词
- 必须包含关键词（通常在摘要末尾另起一行）
- 时态：方法用过去时，结论用现在时
- 必须有具体的数值结果

### 国赛/CUMCM 中文摘要
- 长度：300-500 字
- 包含关键词（3-5 个）
- 必须体现模型特色和创新点
- 结果部分要有具体数据支撑

## 生成步骤

1. **收集素材**：
   - 读取所有 `mm-workspace/03_task_{id}.json`
   - 提取每个任务的 `result_interpretation` 和 `answer`
   - 提取关键数值结果

2. **提取关键信息**：
   - 问题的核心挑战是什么？
   - 采用了哪些主要方法？（最多列出 3 个）
   - 最关键的 3-5 个数值结果是什么？
   - 模型的创新点是什么？

3. **撰写摘要**：
   - 按结构模板组织
   - 每个句子都有信息量，不写空话
   - 数值结果必须有具体数字和单位
   - 控制在字数限制内

4. **质量检查**：
   - 是否包含具体数值结果？（必须有）
   - 是否体现了方法创新？
   - 字数是否在限制内？
   - 是否有语法或拼写错误？

## 保存格式

摘要保存到 `mm-workspace/04_abstract.json`：

```json
{
  "abstract_text": "摘要正文",
  "keywords": ["关键词1", "关键词2", "关键词3"],
  "language": "en 或 zh",
  "word_count": 250,
  "stage": "abstract_complete"
}
```
